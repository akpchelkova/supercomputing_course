основная программа:
- balance_nonblocking.c - программа для исследования совмещения вычислений и коммуникаций
  с использованием неблокирующих операций mpi. измеряет эффективность overlap'а и определяет
  оптимальное соотношение вычислений к коммуникациям.

вспомогательные скрипты:
- run_mpi7_experiments.sh - комплексный скрипт для запуска серии экспериментов с разными параметрами
- test_overlap.sh - базовый скрипт для тестирования совмещения на разных конфигурациях

алгоритм работы:

1. компиляция программы:
   mpicc balance_nonblocking.c -o balance_nonblocking -lm

2. запуск экспериментов:
   - комплексное тестирование: ./run_mpi7_experiments.sh
   - базовое тестирование совмещения: ./test_overlap.sh

3. как работает программа:
   - использует неблокирующие операции MPI_Isend и MPI_Irecv
   - запускает коммуникации ДО начала вычислений для их совмещения
   - выполняет вычисления (usleep) параллельно с передачей данных
   - ждет завершения коммуникаций после вычислений
   - измеряет отдельно время вычислений и коммуникаций
   - вычисляет эффективность совмещения

4. ключевые особенности неблокирующих операций:
   - MPI_Isend: немедленно возвращает управление, отправка происходит в фоне
   - MPI_Irecv: немедленно возвращает управление, прием происходит в фоне  
   - MPI_Waitall: блокируется пока операции не завершатся
   - позволяют совмещать вычисления с коммуникациями

5. метрики эффективности:
   - overlap efficiency: эффективность совмещения в процентах
   - communication-bound: когда коммуникации занимают больше времени чем вычисления
   - compute-bound: когда вычисления занимают больше времени чем коммуникации

получаемые данные:
- время вычислений и коммуникаций для разных конфигураций
- эффективность совмещения вычислений и коммуникаций
- определение типа ограничения производительности (compute-bound vs communication-bound)
- рекомендации по оптимизации соотношения вычислений/коммуникаций

анализ результатов:
- позволяет определить оптимальное соотношение вычислений к коммуникациям
- показывает когда неблокирующие операции эффективны, а когда нет
- демонстрирует влияние количества процессов на возможность совмещения
- помогает найти точку где коммуникации полностью скрываются за вычислениями

практические выводы:
- неблокирующие операции эффективны когда вычисления достаточно длительные
- при малом объеме вычислений коммуникации не успевают скрыться
- оптимальное соотношение зависит от характеристик сети и вычислительной системы
- эффективность совмещения падает с ростом количества процессов из-за увеличения коммуникаций
