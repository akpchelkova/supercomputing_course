1. компиляция программы:
   - выполняем команду: mpicc -o2 derived_types.c -o derived_types
   - это компилирует программу с оптимизацией уровня 2

2. запуск тестов:
   - основной скрипт: ./run_mpi10_comprehensive.sh
   - скрипт автоматически выполняет три конфигурации:
     * 2 процесса на 1 узле (внутриузловая коммуникация)
     * 4 процесса на 2 узлах (межузловая коммуникация)
     * 8 процессов на 4 узлах (распределенная коммуникация)

3. что тестируется:
   - три метода передачи структурированных данных:
     * производные типы данных mpi
     * ручная упаковка/распаковка с mpi_pack/mpi_unpack
     * раздельная передача каждого поля структуры
   - четыре размера данных: 1, 10, 100, 1000 структур Particle
   - каждая комбинация тестируется 100 раз для точности измерений

4. структура данных:
   - Particle содержит: int id, double values[3], char name[20]
   - общий размер: 4 + 24 + 20 = 48 байт (с учетом выравнивания)

5. реализация методов:
   - производные типы: создается mpi_datatype для всей структуры, передача одним сообщением
   - упаковка: данные вручную упаковываются в буфер, передается одно упакованное сообщение
   - раздельная передача: каждое поле передается отдельным mpi сообщением

6. дополнительные тесты:
   - контигуозный тип для массива из 5 double
   - измерение времени для 1000 итераций

7. результаты:
   - выводятся в консоль для каждого теста
   - показывается абсолютное время и отношение времени к производным типам
   - ratio > 1: метод медленнее производных типов
   - ratio < 1: метод быстрее производных типов

8. анализ эффективности:
   - производные типы: минимальные накладные расходы, одно сообщение
   - упаковка: дополнительные затраты на упаковку/распаковку, но одно сообщение
   - раздельная передача: максимальные накладные расходы, много сообщений
