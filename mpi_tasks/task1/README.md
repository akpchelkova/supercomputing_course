основная программа:
- min_max.c - mpi-программа для поиска минимального и максимального значения в векторе. 
  проводит серию экспериментов с разным количеством процессов и разными размерами вектора,
  измеряя время выполнения для каждого случая.

вспомогательные скрипты для запуска на кластере:
- job.sh - основной скрипт для запуска программы с 8 процессами на 1 узле
- test_processes.sh - тестирует производительность при разном количестве процессов (1, 2, 4, 8, 16) 
  с фиксированным размером вектора 1 млн элементов
- test_sizes.sh - тестирует производительность при разных размерах вектора (от 1 тыс до 10 млн элементов) 
  с фиксированным количеством процессов (8)
- multinode.sh - тестирует производительность при распределении процессов по разным узлам кластера

алгоритм работы:

1. компиляция программы:
   mpicc min_max.c -o min_max

2. запуск основных тестов (в порядке рекомендуемого выполнения):
   - базовый тест: sbatch job.sh
   - тест разного количества процессов: sbatch test_processes.sh
   - тест разных размеров данных: sbatch test_sizes.sh
   - тест многоУЗЛОВой конфигурации: sbatch multinode.sh

3. как работает программа:
   - процесс 0 генерирует случайный вектор заданного размера
   - вектор разбивается на части и распределяется между процессами
   - каждый процесс находит локальные min/max в своей части вектора
   - с помощью mpi_reduce находятся глобальные min/max из всех локальных
   - измеряется и выводится время выполнения для каждой конфигурации

получаемые данные:
- время выполнения для разных комбинаций (процессы × размер_вектора)
- значения минимума и максимума для проверки корректности
- данные позволяют анализировать ускорение от распараллеливания
- показывают эффективность работы при разном количестве процессов и объеме данных
- демонстрируют масштабируемость на multiple узлах кластера
