основные программы:
- matrix_band_all_modes.c - программа умножения матриц с поддержкой четырех режимов передачи mpi.
  тестирует стандартный, синхронный, готовностный и буферизованный режимы на одной вычислительной задаче.

- matrix_buffered_only.c - специализированная версия только для буферизованного режима.
  используется для детального тестирования буферизованных операций.

вспомогательные скрипты:
- run_complete_modes_test.sh - запускает полное сравнение всех четырех режимов передачи
- test_buffered.sh - тестирует только буферизованный режим для проверки его работы

алгоритм работы:

1. компиляция программ:
   mpicc matrix_band_all_modes.c -o matrix_band_all_modes
   mpicc matrix_buffered_only.c -o matrix_buffered

2. запуск тестов:
   - полное сравнение всех режимов: ./run_complete_modes_test.sh
   - тестирование только буферизованного режима: ./test_buffered.sh

3. как работают программы:
   - используют ленточный алгоритм умножения матриц из задания 4
   - матрица A распределяется по строкам между процессами
   - матрица B полностью копируется на все процессы
   - каждый процесс умножает свою часть A на всю матрицу B
   - результаты собираются на главном процессе

4. тестируемые режимы передачи:
   - стандартный (MPI_Send): может быть буферизованным или синхронным в зависимости от реализации
   - синхронный (MPI_Ssend): блокируется пока получатель не начнет прием
   - готовностный (MPI_Rsend): предполагает что получатель уже ждет сообщение
   - буферизованный (MPI_Bsend): данные копируются в буфер и функция возвращается сразу

5. особенности реализации:
   - для буферизованного режима все процессы должны иметь присоединенный буфер
   - для готовностного режима требуется синхронизация перед отправкой
   - размер буфера рассчитывается исходя из размера матрицы
   - все режимы используют одинаковый алгоритм вычислений для чистоты эксперимента

получаемые данные:
- время выполнения умножения матриц для каждого режима передачи
- сравнение производительности четырех режимов mpi
- зависимость времени от размера матрицы и количества процессов
- данные о корректности работы разных режимов передачи

анализ результатов:
- позволяет определить самый эффективный режим передачи для данной задачи
- показывает влияние размера сообщений на производительность разных режимов
- демонстрирует trade-off между надежностью и производительностью передачи
- помогает выбрать оптимальный режим передачи для конкретного сценария использования

ожидаемые результаты:
- буферизованный режим: быстрее для больших сообщений, но требует памяти для буфера
- синхронный режим: надежнее, но может быть медленнее из-за блокировок
- готовностный режим: самый быстрый при правильной синхронизации, но может приводить к ошибкам
- стандартный режим: баланс между производительностью и надежностью
