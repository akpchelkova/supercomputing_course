основные программы:
- message_exchange.c - программа для измерения производительности обмена сообщениями между процессами.
  тестирует зависимость времени передачи от размера сообщения, вычисляет пропускную способность,
  проверяет корректность данных и тестирует межУЗЛОВую связь.

- message_exchange_advanced.c - расширенная версия программы, которая сравнивает три разных режима 
  mpi-передачи: синхронный (ssend), готовностный (rsend) и буферизованный (bsend).

вспомогательные скрипты для запуска на кластере:
- job.sh - запускает расширенное тестирование на 2 узлах с 4 процессами
- job_varying.sh - тестирует базовую программу с разным количеством процессов (2, 4, 8, 16)

алгоритм работы:

1. компиляция программ:
   mpicc message_exchange.c -o message_exchange
   mpicc message_exchange_advanced.c -o message_exchange_advanced

2. запуск тестов:
   - базовое тестирование с разным количеством процессов: sbatch job_varying.sh
   - расширенное тестирование режимов передачи: sbatch job.sh

3. как работают программы:
   - процесс 0 и процесс 1 многократно обмениваются сообщениями (1000 раз для усреднения)
   - для каждого размера сообщения замеряется время выполнения
   - вычисляется среднее время на один обмен и пропускная способность
   - проверяется корректность переданных данных
   - в расширенной версии тестируются три режима mpi-передачи

4. тестируемые режимы передачи:
   - синхронный (MPI_Ssend): блокируется пока получатель не начнет прием
   - готовностный (MPI_Rsend): предполагает что получатель уже ждет сообщение
   - буферизованный (MPI_Bsend): данные копируются в буфер и функция возвращается сразу

получаемые данные:
- таблица зависимости времени передачи от размера сообщения
- пропускная способность канала связи в МБ/с
- сравнение производительности разных режимов передачи
- данные о межУЗЛОВой и внутриУЗЛОВой коммуникации
- информация о распределении процессов по узлам кластера

анализ результатов:
- позволяет определить латентность сети (время для маленьких сообщений)
- показывает пропускную способность сети для больших сообщений
- помогает выбрать оптимальный режим передачи для разных сценариев
- демонстрирует разницу в производительности внутриУЗЛОВой и межУЗЛОВой связи
