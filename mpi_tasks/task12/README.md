1. компиляция программы:
   - выполняем команду: mpicc -o3 topology_benchmark.c -o topology_benchmark
   - используется оптимизация уровня 3 для максимальной производительности

2. запуск тестов:
   - основной скрипт: ./run_experiments.sh
   - скрипт автоматически создает директорию для результатов с временной меткой
   - тестируются три конфигурации: 2, 4, 8 процессов

3. что тестируется:
   - четыре типа топологий mpi:
     * декартова топология (2d решетка)
     * топология тора (периодическая решетка)
     * топология графа (произвольная связность)
     * топология звезды (централизованная)
   - измеряется время создания каждой топологии
   - проводится сравнительный анализ производительности

4. реализация топологий:

   декартова топология:
   - используется MPI_Dims_create для автоматического определения размеров решетки
   - MPI_Cart_create создает 2d решетку с непериодическими границами
   - процессы организуются в прямоугольную сетку

   топология тора:
   - используется та же решетка, но с периодическими границами
   - MPI_Cart_create с periods[2] = {1, 1}
   - крайние процессы соединяются с противоположными краями

   топология графа:
   - используется MPI_Graph_create для произвольной связности
   - создается простой линейный граф 0-1-2
   - требуется минимум 3 процесса для работы

   топология звезды:
   - также реализуется через MPI_Graph_create
   - процесс 0 является центральным узлом
   - все остальные процессы соединены только с центральным

5. измерения производительности:
   - используется MPI_Wtime() для точного измерения времени
   - измеряется только время создания топологии (без передачи данных)
   - результаты сохраняются в csv-подобном формате

6. анализ результатов:
   - автоматически генерируется итоговый отчет
   - сравнивается время создания разных топологий
   - проверяется корректность реализации всех требуемых топологий

7. особенности реализации:
   - корректное освобождение коммуникаторов через MPI_Comm_free
   - обработка граничных случаев (недостаточное количество процессов)
   - синхронизация процессов перед измерениями
   - вывод информации только от корневого процесса

8. верификация выполнения:
   - скрипт проверяет успешность выполнения каждого теста
   - генерируется отчет о выполнении всех требований задания
   - подтверждается реализация всех четырех типов топологий
